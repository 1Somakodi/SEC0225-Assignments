# SEC0225 Assignments Repository
Welcome to the SEC0225 Software Engineering Assignments Repository.

This repository serves as the central location for all coding assignments, practice exercises, and programming challenges issued throughout the SEC0225 Software Engineering course. The assignments are designed to reinforce concepts covered during lessons and provide opportunities to develop practical programming and problem-solving skills.

## About This Repository

Software engineering is a hands-on discipline that is best learned through consistent practice. The exercises in this repository will help you:

* Apply concepts learned during class.
* Strengthen your programming fundamentals.
* Develop logical thinking and problem-solving skills.
* Gain experience using industry-standard tools such as Git and GitHub.
* Build a portfolio of work that demonstrates your progress throughout the course.

Students are expected to complete each assignment independently unless otherwise instructed.

## Assignment Workflow
For every assignment provided in this repository, follow the process below:

### 1. Read the Assignment Carefully
Before writing any code:
* Understand the problem statement.
* Identify the required inputs and outputs.
* Break the problem into smaller tasks.
* Ask questions if any requirements are unclear.

### 2. Create Your Solution
Develop your solution using the programming concepts covered in class.
Examples of concepts you may use include:
* Variables and data types
* Input and output
* Conditional statements (`if`, `elif`, `else`)
* Loops (`for` and `while`)
* Functions
* Lists and dictionaries
* File handling

As the course progresses, assignments will incorporate more advanced concepts.

### 3. Test Your Program
Before submitting your work:
* Run your program multiple times.
* Test different inputs.
* Check for errors and unexpected behavior.
* Ensure your program meets all assignment requirements.
A software engineer's job is not only to write code but also to verify that the code works correctly.

### 4. Commit Your Work
Use Git to track your progress.
Make regular commits with meaningful commit messages such as:

```bash
git commit -m "Completed grade analyzer assignment"
```

Avoid commit messages such as:

```bash
git commit -m "update"
git commit -m "work"
git commit -m "changes"
```
Clear commit messages help document your development process.

### 5. Push to GitHub
Upload your completed work to your personal GitHub repository.
Your repository should contain:
* Source code files
* Any required supporting files
* A README file (where applicable)
* Clear commit history

## Assignment Submission
Once you have completed an assignment:
1. Push your final code to GitHub.
2. Copy the URL of your repository.
3. Share the repository link with your instructor through the designated submission channel.

Example:

```text
https://github.com/yourusername/sec0225-assignment-01
```

Please ensure that:
* The repository link is correct.
* The repository contains all required files.
* The instructor can access the repository.

## Recommended Repository Structure

A typical assignment repository should be organized as follows:

```text
sec0225-assignment-01/
│
├── README.md
├── main.py
└── additional_files/
```

Keeping your repository organized makes it easier for others to review your work and demonstrates professionalism.

## Evaluation Criteria

Assignments may be evaluated based on:
* Correctness of the solution
* Code readability
* Proper use of programming concepts
* Problem-solving approach
* Program output
* Repository organization
* Git commit history
* Adherence to assignment instructions

Writing code that works is important, but writing clean, understandable, and maintainable code is equally valuable.

## Academic Integrity
Students are encouraged to collaborate in discussions and support one another in understanding concepts. However, all submitted code must represent your own work.
The following are considered unacceptable:
* Copying another student's code.
* Submitting work found online as your own.
* Sharing complete solutions with classmates.

Learning happens when you solve problems yourself, make mistakes, and improve from them.

---

## Getting Help
If you get stuck while working on an assignment:
1. Review your class notes.
2. Revisit the lesson materials.
3. Read error messages carefully.
4. Debug your code step by step.
5. Seek clarification from your instructor or teaching assistants.
Remember: encountering challenges is a normal part of learning to program.
